Background

Continuing the work from the original 2015 proposal for Parinfer.

Parinfer was designed to help newcomers with Lisp. It allows Lisp code to be driven by indentationâ€”not by removing Parentheses, but by inferring them. It now exists as a plugin across many editors, and is reportedly used by newcomers and experienced users across different Lisps.

Some parts didn't feel right over time. Though many of us enjoyed the core of its behavior, some edge cases had emerged to cause unintended structure changes. And as we anticipated, users expressed problems with mode-switch confusion and noisy diffs on projects.

We'd like to show how it has improved. There is now an easier unified mode, a project linter for teams, and some safeguards to prevent unintended structure changes. Not all problems are solved, but this represents a milestone toward discovering more of Parinfer capabilities and tradeoffs so that we may better understand how to improve the newcomer experience for Lisp.
